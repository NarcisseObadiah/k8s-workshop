Core concepts
PodSmallest deployable unit. One or more containers sharing network & storage.
DeploymentDeclares desired state. Manages replicas, rolling updates, and rollbacks.
ReplicaSetKeeps N identical pods running. Managed automatically by Deployment.
ServiceStable network endpoint. Routes traffic to pods via label selectors.
NamespaceVirtual cluster for isolation — scopes resources, quotas, and access.
NodeWorker machine (VM or bare metal) that runs pods.
etcdKey-value store. Single source of truth for all cluster state.
kubeletAgent on each node. Executes pod specs, reports health to API server.
SchedulerPicks which node fits a new pod based on resources & constraints.
Controller mgrReconciliation loop — constantly drives actual state toward desired state.
Cluster architecture
Control plane
  • API server
  • Scheduler
  • Controller manager
  • etcd
Worker node
  • kubelet
  • kube-proxy
  • Container runtime
  • Pods
Imperative vs declarative
Imperative — quick experiments
kubectl create deployment ...
kubectl scale ... --replicas=4
kubectl expose ...
Declarative — production & teams
Write YAML manifest
kubectl apply -f deploy.yaml
Git commit = source of truth

With apply, Kubernetes diffs the current state against your manifest and makes only the necessary changes — safe to re-run at any time.

Resource requests & limits
resources: requests: cpu: "100m" # scheduler uses this memory: "128Mi" limits: cpu: "250m" # hard cap — throttled memory: "256Mi" # hard cap — OOMKilled
100m CPU0.1 of a core. 1000m = 1 full core.
requestsUsed by scheduler to find a node with enough capacity.
limitsRuntime enforcement. CPU = throttled. Memory = pod killed.
Namespaces
kubectl get nsList all namespaces in the cluster.
-n <ns>Scope any command to a specific namespace.
--all-namespacesShow resources across every namespace.
delete namespaceDeletes the namespace and everything inside it.

Default namespaces: default, kube-system, kube-public. Real clusters use per-team or per-env namespaces with quotas and access controls.

Lab steps  (75 min)
  • 1
    Deploy — imperative Get the app running fast with kubectl.
    kubectl create deployment k8s-demo --image=ghcr.io/narcisseobadiah/k8s-workshop-demo:latest --port=3000
  • 2
    Declarative YAML Delete the deployment, rewrite as a manifest with resource requests & limits, apply it.
    kubectl delete deployment k8s-demo
    kubectl apply -f deployment.yaml
  • 3
    Namespaces Create a workshop namespace, deploy there, compare visibility, clean up.
    kubectl create namespace workshop
    kubectl apply -f deployment.yaml -n workshop
    kubectl get pods --all-namespaces
  • 4
    Expose as Service Add a NodePort Service to the manifest, apply, open in KillerKoda traffic panel.
    kubectl apply -f deployment.yaml && kubectl get svc k8s-demo
  • 5
    Scale & self-heal Scale to 4 replicas via YAML, delete a pod, watch Kubernetes replace it.
    kubectl apply -f deployment.yaml  # after editing replicas: 4
    kubectl delete pod <name> && kubectl get pods -w
  • 6
    Rolling update Trigger an image update, watch pods replaced one-by-one, check history.
    kubectl set image deployment/k8s-demo k8s-workshop-demo=...latest
    kubectl rollout status deployment/k8s-demo

What happened under the hood
create / applyAPI server saved desired state to etcd; scheduler placed pod; kubelet pulled image
resources.requestsScheduler used these to find a node with enough free capacity
resources.limitskubelet enforces at runtime — CPU throttled, memory OOMKills the pod
namespace createdLogical boundary — own quota, network policy, and RBAC scope
expose / Servicekube-proxy set up rules to forward traffic to any matching pod
delete podController detected actual < desired; replacement pod created in seconds
set imageNew ReplicaSet created; pods replaced one-by-one — zero downtime
What to explore next
ConfigMap Inject config into pods without rebuilding images.
Secret Store passwords and tokens — base64 encoded, RBAC controlled.
Liveness probe Kubernetes restarts pods that fail a health check endpoint.
Ingress HTTP routing — hostnames, paths, TLS termination.
Helm Package manager — deploy complex apps as versioned charts.
HPA Horizontal Pod Autoscaler — scale automatically on CPU/memory.
